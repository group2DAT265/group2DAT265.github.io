\doxysubsection{Detailed Description}
This is an immutable class representing information of either {\ttfamily author} or {\ttfamily editor} field in bibtex record. 

Constructor performs parsing of raw field text and stores preformatted data. Various accessor methods return author/editor field in different formats. 

Parsing algorithm is designed to satisfy two requirements\+: (a) when author\textquotesingle{}s name is typed correctly, the result should coincide with the one of Bi\+B\+TeX; (b) for erroneous names, output should be reasonable (but may differ from Bi\+B\+TeX output). The following rules are used\+: 
\begin{DoxyEnumerate}
\item \textquotesingle{}author field\textquotesingle{} is a sequence of tokens; 
\begin{DoxyItemize}
\item tokens are separated by sequences of whitespaces ({\ttfamily Character.\+is\+Whitespace(c)==true}), commas (,), dashes (-\/), and tildas ($\sim$); 
\item every comma separates tokens, while sequences of other separators are equivalent to a single separator; for example\+: \char`\"{}a -\/ b\char`\"{} consists of 2 tokens (\char`\"{}a\char`\"{} and \char`\"{}b\char`\"{}), while \char`\"{}a,-\/,b\char`\"{} consists of 3 tokens (\char`\"{}a\char`\"{}, \char`\"{}\char`\"{}, and \char`\"{}b\char`\"{}) 
\item anything enclosed in braces belongs to a single token; for example\+: \char`\"{}abc x\{a,b,-\/$\sim$ c\}x\char`\"{} consists of 2 tokens, while \char`\"{}abc xa,b,-\/$\sim$ cx\char`\"{} consists of 4 tokens (\char`\"{}abc\char`\"{}, \char`\"{}xa\char`\"{},\char`\"{}b\char`\"{}, and \char`\"{}cx\char`\"{}); 
\item a token followed immediately by a dash is \char`\"{}dash-\/terminated\char`\"{} token, and all other tokens are \char`\"{}space-\/terminated\char`\"{} tokens; for example\+: in \char`\"{}a-\/b-\/ c -\/ d\char`\"{} tokens \char`\"{}a\char`\"{} and \char`\"{}b\char`\"{} are dash-\/terminated and \char`\"{}c\char`\"{} and \char`\"{}d\char`\"{} are space-\/terminated; 
\item for the purposes of splitting of \textquotesingle{}author name\textquotesingle{} into parts and construction of abbreviation of first name, one needs definitions of first letter of a token, case of a token, and abbreviation of a token\+: 
\begin{DoxyItemize}
\item \textquotesingle{}first letter\textquotesingle{} of a token is the first letter character ({\ttfamily Character.\+is\+Letter(c)==true}) that does not belong to a sequence of letters that immediately follows \char`\"{}\textbackslash{}\char`\"{} character, with one exception\+: if \char`\"{}\textbackslash{}\char`\"{} is followed by \char`\"{}aa\char`\"{}, \char`\"{}\+A\+A\char`\"{}, \char`\"{}ae\char`\"{}, \char`\"{}\+A\+E\char`\"{}, \char`\"{}l\char`\"{}, \char`\"{}\+L\char`\"{}, \char`\"{}o\char`\"{}, \char`\"{}\+O\char`\"{}, \char`\"{}oe\char`\"{}, \char`\"{}\+O\+E\char`\"{}, \char`\"{}i\char`\"{}, or \char`\"{}j\char`\"{} followed by non-\/letter, the \textquotesingle{}first letter\textquotesingle{} of a token is a letter that follows \char`\"{}\textbackslash{}\char`\"{}; for example\+: in \char`\"{}a\{x\}b\char`\"{} \textquotesingle{}first letter\textquotesingle{} is \char`\"{}a\char`\"{}, in \char`\"{}\{\textbackslash{}\char`\"{}\{U\}\}bel\char`\"{} \textquotesingle{}first letter\textquotesingle{} is \char`\"{}U\char`\"{}, in
\char`\"{}\{\textbackslash{}noopsort\{"{}o\}\}xyz\char`\"{} \textquotesingle{}first letter\textquotesingle{} is \char`\"{}o\char`\"{}, in \char`\"{}\{\textbackslash{}\+AE\}x\char`\"{} \textquotesingle{}first letter\textquotesingle{} is
\char`\"{}A\char`\"{}, in \char`\"{}\textbackslash{}aex\textbackslash{}ijk\textbackslash{}\+Oe\textbackslash{}j\char`\"{} \textquotesingle{}first letter\textquotesingle{} is \char`\"{}j"{}; if there is no letter satisfying the above rule, \textquotesingle{}first letter\textquotesingle{} is undefined; 
\item token is \char`\"{}lower-\/case\char`\"{} token if its first letter is defined and is lower-\/case ({\ttfamily Character.\+is\+Lower\+Case(c)==true}), and token is \char`\"{}upper-\/case\char`\"{} token otherwise; 
\item \textquotesingle{}abbreviation\textquotesingle{} of a token is the shortest prefix of the token that (a) contains \textquotesingle{}first letter\textquotesingle{} and (b) is braces-\/balanced; if \textquotesingle{}first letter\textquotesingle{} is undefined, \textquotesingle{}abbreviation\textquotesingle{} is the token itself; in the above examples, \textquotesingle{}abbreviation\textquotesingle{}s are \char`\"{}a\char`\"{}, \char`\"{}\{\textbackslash{}\char`\"{}\{U\}\}\char`\"{}, \char`\"{}\{\textbackslash{}noopsort\{"{}o\}\}\char`\"{}, \char`\"{}\{\textbackslash{}\+AE\}\char`\"{},
\char`\"{}\textbackslash{}aex\textbackslash{}ijk\textbackslash{}\+Oe\textbackslash{}j"{}; 
\end{DoxyItemize}
\item the behavior based on the above definitions will be erroneous only in one case\+: if the first-\/name-\/token is \char`\"{}\{\textbackslash{}noopsort\{\+A\}\}john\char`\"{}, we abbreviate it as \char`\"{}\{\textbackslash{}noopsort\{\+A\}\}.\char`\"{}, while Bi\+B\+TeX produces \char`\"{}j.\char`\"{}; fixing this problem, however, requires processing of the preabmle; 
\end{DoxyItemize}
\item \textquotesingle{}author names\textquotesingle{} in \textquotesingle{}author field\textquotesingle{} are subsequences of tokens separated by token \char`\"{}and\char`\"{} (\char`\"{}and\char`\"{} is case-\/insensitive); if \textquotesingle{}author name\textquotesingle{} is an empty sequence of tokens, it is ignored; for examle, both \char`\"{}\+John Smith and Peter

\begin{DoxyItemize}
\item no commas, all tokens are upper-\/case\+: \textquotesingle{}junior-\/part\textquotesingle{} and \textquotesingle{}von-\/part\textquotesingle{} are empty, \textquotesingle{}last-\/part\textquotesingle{} consist of the last token, \textquotesingle{}first-\/part\textquotesingle{} consists of all other tokens (\textquotesingle{}first-\/part\textquotesingle{} is empty if \textquotesingle{}author name\textquotesingle{} consists of a single token); for example, in \char`\"{}\+John James Smith\char`\"{}, \textquotesingle{}last-\/part\textquotesingle{}=\char`\"{}\+Smith\char`\"{} and \textquotesingle{}first-\/part\textquotesingle{}=\char`\"{}\+John James\char`\"{};
