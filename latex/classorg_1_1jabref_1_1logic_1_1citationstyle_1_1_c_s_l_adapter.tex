\doxysubsection{Detailed Description}
Provides an adapter class to C\+SL. It holds a C\+SL instance under the hood that is only recreated when the style changes.

@api\+Note The first call to \mbox{\hyperlink{classorg_1_1jabref_1_1logic_1_1citationstyle_1_1_c_s_l_adapter_a8711addfcc2fdc4da0c323fd27745286}{make\+Bibliography}} is expensive since the C\+SL instance will be created. As long as the style stays the same, we can reuse this instance. On style-\/change, the engine is re-\/instantiated. Therefore, the use-\/case of this class is many calls to \mbox{\hyperlink{classorg_1_1jabref_1_1logic_1_1citationstyle_1_1_c_s_l_adapter_a8711addfcc2fdc4da0c323fd27745286}{make\+Bibliography}} with the same style. Changing the output format is cheap. @impl\+Note The main function \mbox{\hyperlink{classorg_1_1jabref_1_1logic_1_1citationstyle_1_1_c_s_l_adapter_a8711addfcc2fdc4da0c323fd27745286}{make\+Bibliography}} will enforce synchronized calling. The main C\+SL engine under the hood is not thread-\/safe. Since this class is usually called from a Background\+Takk, the only other option would be to create several C\+SL instances which is wasting a lot of resources and very slow. In the current scheme, \mbox{\hyperlink{classorg_1_1jabref_1_1logic_1_1citationstyle_1_1_c_s_l_adapter_a8711addfcc2fdc4da0c323fd27745286}{make\+Bibliography}} can be called as usual background task and to the best of my knowledge, concurrent calls will pile up and processed sequentially.
