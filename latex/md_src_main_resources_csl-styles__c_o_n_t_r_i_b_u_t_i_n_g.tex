The C\+SL style repository is the central location on the web for collecting and maintaining C\+SL styles. Software products like Zotero, Mendeley, and Papers all draw their styles from our repository.

We welcome style submissions (and corrections), and are particularly interested in styles for journals and published style guides. If you wish to submit a different type of style, please first check our \href{https://github.com/citation-style-language/styles/wiki/Criteria-for-Accepting-Styles}{\texttt{ Criteria for Accepting Styles}}.

Update your pull request}\label{md_src_main_resources_csl-styles__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md67}
We often request you to make some changes to your pull request. Don\textquotesingle{}t create a new pull request right away if your original submission has issues. It\textquotesingle{}s much easier for us if you update your original pull request instead.

After you create a pull request, our \href{https://github.com/csl-bot}{\texttt{ @csl-\/bot}} will first automatically run a number of tests to check whether your style validates and follows our style requirements. These tests usually finish within a few minutes. If there are problems, \href{https://github.com/csl-bot}{\texttt{ @csl-\/bot}} will leave a comment with the identified errors. If you have trouble understanding these errors, please post a comment in the pull request and ask for help.

We also have a handful of volunteers who review pull requests, usually within a week or two.

To update your pull request\+:


\begin{DoxyEnumerate}
\item Visit the \char`\"{}\+Files changed\char`\"{} tab of your pull request.
\item Click the ellipsis button at the top-\/right of your style, and select \char`\"{}\+Edit file\char`\"{} in the ellipsis menu.
\item Make your changes, and click the \char`\"{}\+Commit changes\char`\"{} button.
\end{DoxyEnumerate}

Any changes made this way will automatically appear in the pull request, and each commit will trigger a new test report.

Once we are satisfied, we will {\bfseries{merge}} your pull request. This can take days or weeks, depending on how busy we are, and on how much back and forth is needed to get your style in acceptable shape.\hypertarget{md_src_main_resources_csl-styles__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md68}{}\doxysubsubsection{5. Make additional changes after merging}\label{md_src_main_resources_csl-styles__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md68}
Once a pull request has been merged, it can no longer be used to submit further changes. So, if you wish to make additional changes, just create a new pull request.

Please always start from the latest version of the style from our repository, because we may have performed maintenance on your style or accepted corrections from other contributors.\hypertarget{md_src_main_resources_csl-styles__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md69}{}\doxysection{Asking for help}\label{md_src_main_resources_csl-styles__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md69}
